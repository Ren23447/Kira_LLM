# Kira LLM

A GPT-style language model built entirely from scratch вЂ” custom transformer architecture, custom tokenizer, custom training pipeline, custom personality system. No borrowed weights. No fine-tuning. Every piece was designed intentionally.

Built by **Julian Riley Hunter (Ren)**.

---

## What Kira Is

Kira is a conversational AI with:
- A **GPT-style decoder-only transformer** with KV-cache inference
- A **character-level tokenizer** built from training data
- A **persistent memory system** (SQLite + FTS5 search)
- A **personality engine** with trait-driven LLM directives
- An **emotional state engine** that evolves through conversation
- A **reflection engine** that generates observations about the user every 20 turns
- A **learning engine** that builds a dynamic user profile over time
- A **web chat interface** served by Flask

When no trained checkpoint exists, Kira falls back to a **fragment assembly engine** вЂ” a sophisticated rule-based response system вЂ” so it always works, even before training.

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

Python 3.10+ required. CUDA is optional but strongly recommended for training.

### 2. Prepare training data

```bash
python prepare_data.py
```

This generates `data/train.txt`, `data/val.txt`, `data/tokenizer.json`, and `data/stats.txt`.

### 3. Train the model

**GPU (recommended):**
```bash
python train.py                      # small model (~6M params), 50k steps
python train.py --config tiny        # ~1M params, faster
python train.py --config medium      # ~25M params, better quality
python train.py --compile            # torch.compile speedup (PyTorch в‰Ґ 2.0)
```

**CPU only:**
```bash
python train_cpu.py                  # tiny model, 20k steps
python train_cpu.py --steps 5000     # quick smoke-test
```

Checkpoints are saved to `checkpoints/`:
- `checkpoints/best.pt` вЂ” best validation loss seen so far
- `checkpoints/final.pt` вЂ” end of training
- `checkpoints/ckpt_XXXXXXX.pt` вЂ” periodic numbered checkpoints (rotated, keeps last 5)

### 4. Run the chat interface

```bash
python kira_server.py
```

Open your browser to `http://localhost:5000`. Kira automatically loads the best checkpoint if one exists.

### 5. CLI chat (no server)

```bash
python kira.py
```

### 6. Direct inference

```bash
python generate.py --prompt "tell me about black holes"
python generate.py                          # interactive mode
python generate.py --checkpoint checkpoints/best.pt
```

## Memory System

Kira remembers things across sessions using SQLite:

- `user_fact` вЂ” name, personal details
- `project` вЂ” things the user is building
- `goal` вЂ” stated aims and ambitions
- `preference` вЂ” likes and dislikes
- `event` вЂ” things that happened
- `long_term` вЂ” reflections and observations

Memory API endpoints (when server is running):

```
GET    /memory/search?q=robots     search memories
POST   /memory/save                save a memory manually
DELETE /memory/delete/<id>         delete a memory
POST   /memory/consolidate         merge near-duplicates
GET    /profile                    full user profile JSON
GET    /profile/summary            one-paragraph summary
POST   /profile/rebuild            force profile rebuild
```

Copy `.env.example` to `.env` and adjust as needed:

- `KIRA_CHECKPOINT` вЂ” path to checkpoint (default: `checkpoints/best.pt`)
- `KIRA_TEMPERATURE` вЂ” sampling temperature (default: 0.8)

## Architecture Notes

- **Tokenizer**: character-level with 5 special tokens (`<pad>`, `<usr>`, `<kir>`, `<eos>`, `<unk>`). Vocabulary is built from the training data вЂ” no fixed vocab size.
- **Attention**: multi-head causal self-attention with optional flash attention (PyTorch в‰Ґ 2.0) and manual causal mask fallback.
- **KV cache**: inference reuses key/value tensors from previous steps вЂ” O(1) per token rather than O(n).
- **Weight tying**: the token embedding matrix and LM head share weights, reducing parameter count.
- **Training**: cosine LR schedule with linear warm-up, gradient accumulation, AMP (automatic mixed precision), and periodic checkpoint rotation.
- **Fallback**: if no checkpoint exists, `kira.py` uses fragment assembly вЂ” a deterministic, anti-repeat response engine with broad knowledge coverage.
